# Contributing to NebulusDB

Thank you for your interest in contributing to NebulusDB! We welcome contributions from everyone—whether you're fixing bugs, adding features, improving documentation, or helping others.

---

## Getting Started
- **Fork the repository** and clone your fork.
- **Install dependencies:**
  ```sh
  npm install
  ```
- **Run tests:**
  ```sh
  npm test
  ```
- **Read the [ROADMAP.md](./ROADMAP.md)** to see what's planned and where you can help most.

---

## Code Standards
- **TypeScript-first:** Write all code in TypeScript.
- **Linting:** Run `npm run lint` and fix any issues before submitting.
- **Formatting:** Use Prettier (`npm run format`) for consistent code style.
- **Tests:** Add or update tests for any code changes. All tests must pass.
- **Documentation:** Update or add documentation for new features or changes.

---

## Submitting Issues
- **Search first:** Check if your issue or feature request already exists.
- **Bug reports:** Include clear steps to reproduce, expected vs. actual behavior, and environment info.
- **Feature requests:** Explain your use case and why it would benefit NebulusDB.

---

## Pull Requests
- **One feature/fix per PR:** Keep changes focused and easy to review.
- **Describe your changes:** Use clear commit messages and PR descriptions.
- **Link issues:** Reference related issues in your PR.
- **Be responsive:** Answer questions and make requested changes promptly.

---

## Community Guidelines
- **Be respectful:** Treat everyone with kindness and respect.
- **Collaborate:** Help others, review PRs, and share feedback.
- **Stay constructive:** Focus on solutions and improvement.
- **No spam or self-promotion.**

---

## Need Help?
- Join our community chat (Discord/Slack—see README or website)
- Open a GitHub Discussion or Issue

We're excited to build NebulusDB with you!
